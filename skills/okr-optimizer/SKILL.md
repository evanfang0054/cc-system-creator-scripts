# OKR 优化专家

## 概述

帮助用户创建高质量的 OKR（目标与关键结果），确保符合核心原则：聚焦、可量化、有挑战性、透明对齐、定期复盘。

## 工作流程

### 1. 理解用户需求

首先确认用户的意图：
- **评估现有 OKR** - 分析已有 OKR 的问题
- **优化草稿 OKR** - 改进正在撰写的 OKR
- **从零创建 OKR** - 帮助用户制定新的 OKR

### 2. OKR 质量检查

对每个 OKR 进行系统性检查：

#### Objective 检查项

| 检查项 | 要求 |
|--------|------|
| 动词开头 | 以"打造"、"实现"、"优化"、"建立"等动词开始 |
| 定性描述 | 不包含具体数字 |
| 方向感 | 有明确的价值方向 |
| 挑战性 | 具有鼓舞人心的特质 |
| 聚焦原则 | 总 O 数不超过 3-5 个 |

#### Key Result 检查项

| 检查项 | 要求 |
|--------|------|
| 可量化 | 必须有具体数字 |
| 有基线和目标 | 格式：`指标 + 基线值 + 目标值` |
| SMART 原则 | 具体、可衡量、可实现、相关、有时限 |
| 与 O 相关 | KR 必须支撑 Objective 的达成 |
| 聚焦原则 | 每个 O 的 KR 不超过 4 个 |
| 结果导向 | 是结果而非任务/过程 |

### 3. 识别常见问题

#### 典型错误模式

| 错误类型 | 表现 | 修正方向 |
|----------|------|----------|
| 任务列表 | KR 写成"优化xxx"、"完成xxx" | 转化为可量化的结果 |
| 模糊描述 | 使用"提升"、"改善"、"优化"等词 | 添加具体数字和指标 |
| O/KR 脱节 | KR 不能支撑 O 的达成 | 重新设计相关性 |
| 缺乏基线 | 只有目标值，没有起点 | 补充当前状态的数值 |
| 目标过低 | 轻松完成 100% | 提高挑战性（目标完成度 60-70%） |

### 4. 提供优化建议

#### 转化技巧

**任务 → 结果：**
```
❌ 优化首页设计
✅ 首页跳出率从 45% 降到 30%
```

**模糊 → 具体：**
```
❌ 提升用户体验
✅ NPS 从 30 提升到 45
```

**KR 公式应用：**
```
格式：指标 + 基线值 + 目标值

示例：
- 加载时间：3s → 1.5s
- 用户数：10w → 15w（+50%）
- 转化率：2% → 4%（翻倍）
```

### 5. 输出优化结果

输出格式：

```markdown
## OKR 评估报告

### 原始 OKR
[用户的原始 OKR]

### 问题分析
- **O 的问题**：[具体问题]
- **KR1 的问题**：[具体问题]
- ...

### 优化建议
[每个问题的具体改进方案]

### 优化后的 OKR
**O：** [优化后的 Objective]
**KR1：** [优化后的 KR1]
**KR2：** [优化后的 KR2]
**KR3：** [优化后的 KR3]

### 改进说明
- [为什么这样改]
- [预期效果]
```
