# OKR 核心原则与写作指南

## 一、什么是 OKR

- **O (Objective)**：目标 - 你想达成什么？有挑战性、鼓舞人心的定性描述
- **KR (Key Results)**：关键结果 - 如何衡量目标达成？3-5个可量化的指标

## 二、核心原则

| 原则 | 说明 |
|------|------|
| **聚焦** | 少即是多，O 不超过 3-5 个，每个 O 的 KR 不超过 4 个 |
| **可量化** | KR 必须是数字，不能模糊（如"提升用户体验"❌ "NPS 从 30 提升到 45"✅） |
| **有挑战** | 好的 OKR 完成度在 60-70%，太容易说明不够激进 |
| **透明对齐** | 个人 OKR 要对齐团队/部门 OKR，全公司可见 |
| **定期复盘** | 每周检查进度，季度末评分（0-1分） |

## 三、写作要求

### Objective 写法

- **动词开头**：打造、实现、优化、建立...
- **定性描述，不带数字**
- **有方向感和价值感**

**示例对比：**
- ❌ 不好的例子："完成 Q1 业绩"
- ✅ 好的例子："打造行业内最流畅的用户注册体验"

### KR 写法（SMART 原则）

- **Specific** - 具体明确
- **Measurable** - 可衡量
- **Achievable** - 可实现（有挑战性但非不可能）
- **Relevant** - 与目标强相关
- **Time-bound** - 有时限（默认季度）

**KR 公式：** `指标 + 基线值 + 目标值`

**例如：** DAU 从 10w 提升到 15w（+50%）

## 四、常见错误

| 错误 | 说明 |
|------|------|
| 把 OKR 当成 Todo List | KR 是结果，不是任务 |
| KR 过多 | 聚焦最重要的事 |
| 不敢设高目标 | 完成 100% 说明目标设低了 |
| KR 不可验证 | 避免"提升"、"优化"等模糊词 |
| O 和 KR 脱节 | KR 必须能支撑 O 的达成 |

## 五、检查清单

### Objective 检查项

- [ ] 以动词开头
- [ ] 定性描述，不含数字
- [ ] 有方向感和价值感
- [ ] 具有挑战性和鼓舞人心
- [ ] 不超过 5 个字（聚焦原则）

### Key Result 检查项

- [ ] 可量化（有具体数字）
- [ ] 有基线值和目标值
- [ ] 符合 SMART 原则
- [ ] 与 Objective 强相关
- [ ] 每个 O 的 KR 不超过 4 个
- [ ] 不是任务列表（是结果而非过程）
