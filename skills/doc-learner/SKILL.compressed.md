# 文档学习与内容模仿

## 工作流程

### 1. 文档接收与分类
**支持类型：** 提示词、技术文档、API文档、用户手册、产品说明、其他结构化文档

**确认目标：** 模仿什么？（结构/风格/格式/模式）→ 生成什么？（提示词/文档/模板）

### 2. 深度文档分析
**2.1 结构分析：** 文档组织、信息架构、模块化程度

**2.2 风格分析：** 语言风格、语气表达、术语规范

**2.3 模式识别：** 句式模式、常用句型、标准表达

**2.4 要素提取：** 关键信息点、核心概念、必需组成部分

### 3. 模式提炼与模板创建
**3.1 结构模板：** 文档骨架、章节框架、标准化结构

**3.2 写作模式：** 句式模板库、表达模式集合、风格指南

**3.3 关键要素清单：** 必备内容检查表、核心要素列表、质量标准

### 4. 内容生成与优化
**4.1 内容生成：** 应用学习模式、填充结构模板、应用写作风格

**4.2 质量验证：** 检查模式遵循、验证结构完整性、确保风格一致性

**4.3 迭代优化：** 根据反馈调整、持续改进模板库

## 核心能力

**文档分析引擎（三维度）：**
- 结构：识别文档类型、提取组织架构、分析信息流
- 风格：分析语言特征、识别语气、提取风格标记
- 模式：发现表达模式、提取可重用句式、识别标准表达

**智能模板提取：**
- 结构模板：文档框架、章节模板、段落结构
- 内容模式：句式库、表达模式、术语集合
- 风格指南：语言规范、表达习惯、最佳实践

**内容生成支持：**
- 模板化：填空式创建、结构化组装、模块化复用
- 风格迁移：应用写作风格、模仿表达方式、保持一致性
- 质量保证：模式验证、结构检查、风格评估

## 使用场景

**场景 1：学习优秀提示词**
1. 分析样本 → 2. 识别结构（角色/任务/约束/格式）→ 3. 提炼模式 → 4. 创建模板 → 5. 生成新提示词

**场景 2：模仿技术文档风格**
1. 分析结构风格 → 2. 识别组织方式 → 3. 提炼术语规范 → 4. 创建风格指南 → 5. 指导生成

**场景 3：文档质量提升**
1. 对比分析 → 2. 识别差距 → 3. 应用优秀模式 → 4. 优化文档 → 5. 提供建议

## 最佳实践

1. **多样本学习：** 收集3-10个高质量样本，识别共性模式，提炼稳定有效模式
2. **模式验证：** 在实际应用中验证，根据效果调整优化，持续改进模板库
3. **灵活应用：** 不机械套用模板，根据需求灵活调整，保持创造性思维
4. **质量优先：** 注重内容质量而非形式模仿，确保生成内容有价值

## 常见问题

**Q: 学习多少个样本合适？**
A: 建议 3-10 个高质量样本

**Q: 如何处理不同风格？**
A: 分别学习不同风格，创建多个风格模板

**Q: 生成内容完全模仿原文？**
A: 不，学习应用模式结构，内容本身应原创

**Q: 支持非中文文档？**
A: 支持多语言文档学习和模仿
