# booking页面测试用例

该文档包含了booking页面的测试用例，用于确保验收补全后的胶水代码是否符合测试验收标准

## 测试点

### 前置条件1

姓名、电话、邮箱都没有配置

- 操作步骤
1、选择预定日期和时间，填写航班号
2、点击continue to checkout
- 预期结果
1、跳转至chekcout页面
2、不展示lead passenger和delivery模块

### 前置条件2

姓名、电话、邮箱配置其中一项

- 操作步骤1
1、选择3人
2、选择预定日期和时间，填写航班号
3、点击continue to checkout
- 预期结果1
1、跳转至contact details页面
2、显示3个姓名输入框+1个电话输入框+一个邮箱输入框

- 操作步骤2
1、contact details页填写完所有信息
2、点击continue按钮
- 预期结果2
1、跳转至chekcout页面
2、没填写的字段不显示

### 前置条件3

相关接口请求成功以及返回数据跟组件交互逻辑对接

- 操作步骤1
1、选择服务人数
2、点击选择日期
3、选择可用日期
4、点击可用时间
- 预期结果1
1、可选日期接口重新请求
2、可用日期展示在日历上
3、可选时间接口重新请求
4、时间选择器展示可用时间
