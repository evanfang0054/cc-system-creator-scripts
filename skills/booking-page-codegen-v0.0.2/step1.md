# 第 1 步：需求分析与代码结构

## 目标

分析提供的静态模板，理解完整的业务需求和代码结构。

## 行动

### 1. 分析静态模板代码

- 阅读并理解提供的 `index.tsx` 及相关文件
- 识别现有的 UI 结构、组件层级和 TODO 逻辑点
- 分析 `useBookingStore` 状态定义（如果提供）
- 记录当前实现差距和完成点

### 2. 阅读产品需求

- 完全阅读: [references/prd.md](references/prd.md)
- 理解业务工作流和核心功能：
  - 日期选择逻辑
  - 乘客信息动态渲染
  - 预订策略验证规则
- 识别数据流和字段需求
- 将用户场景映射到技术需求

### 3. 审查编码标准

- 完全阅读: [references/codingspec.md](references/codingspec.md)
- 理解项目约定：
  - 文件命名模式
  - 变量命名标准
  - 注释要求
  - React Hooks 使用指南
  - 组件组织原则

- 业务逻辑已映射

✅ **标准审查**：
- 编码约定已理解
- 项目结构清晰
