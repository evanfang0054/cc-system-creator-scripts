# Mole 工作流模板

本文档提供针对常见场景的标准化工作流，供 Agent 执行时参考。

## ⚠️ 核心原则：扫描 → 展示 → 确认 → 执行

**所有清理操作必须遵循此四步流程，绝不可跳过任何步骤。**

```
┌────────────────────────────────────────────────────────────────────────┐
│                                                                        │
│   第一步：扫描（--dry-run）                                              │
│   ↓                                                                    │
│   第二步：整理并向用户展示清理方案                                        │
│   ↓                                                                    │
│   第三步：使用 AskUserQuestion 询问用户确认                              │
│   ↓                                                                    │
│   第四步：用户明确同意后才执行                                            │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

---

## 工作流 1：磁盘空间不足

**触发条件**：用户报告磁盘空间不足、需要释放空间

### 完整执行步骤

**步骤 1：检查当前状态**
```bash
mo --version
df -h /
```

**步骤 2：扫描系统缓存**
```bash
mo clean --dry-run
```

**步骤 3：整理并展示扫描结果**

必须以清晰格式呈现：
```
📊 系统缓存扫描结果：

┌─────────────────────────────┬───────────┐
│ 清理项目                     │ 大小       │
├─────────────────────────────┼───────────┤
│ 用户应用缓存                  │ XX.X GB   │
│ 浏览器缓存                    │ XX.X GB   │
│ 开发工具缓存                  │ XX.X GB   │
│ 系统日志                     │ XX.X GB   │
│ 废纸篓                       │ XX.X GB   │
├─────────────────────────────┼───────────┤
│ 总计可释放                    │ XX.X GB   │
└─────────────────────────────┴───────────┘

当前磁盘可用：XX GB → 清理后预计：XX GB
```

**步骤 4：询问用户确认**
```
使用 AskUserQuestion 询问：
"以上项目共可释放 XX.X GB 空间，是否执行清理？"
- 确认执行清理
- 取消操作
- 查看详细信息
```

**步骤 5：用户同意后执行**
```bash
mo clean
```

**步骤 6：（可选）扫描项目构建产物**
```bash
mo purge --dry-run
```

展示结果并询问确认后执行：
```bash
mo purge
```

**步骤 7：（可选）扫描安装包**
```bash
mo installer --dry-run
```

展示结果并询问确认后执行：
```bash
mo installer
```

**步骤 8：报告最终结果**
```
✅ 清理完成！

系统缓存释放：XX GB
项目构建产物释放：XX GB
安装包释放：XX GB
─────────────────
总计释放：XX GB

当前磁盘可用空间：XX GB
```

## 工作流 2：系统变慢/卡顿

**触发条件**：用户报告 Mac 运行缓慢、卡顿

### 完整执行步骤

**步骤 1：检查系统状态**
```bash
mo status
```
观察健康分数、CPU/内存/磁盘使用情况。

**步骤 2：分析问题**
- CPU 占用高 → 检查进程
- 内存不足 → 检查内存使用
- 磁盘快满 → 执行工作流 1

**步骤 3：扫描缓存**
```bash
mo clean --dry-run
```

**步骤 4：展示扫描结果并询问确认**
```
📊 发现可清理缓存：XX.X GB

是否执行清理？
- 确认清理
- 取消
```

**步骤 5：用户同意后执行**
```bash
mo clean
```

**步骤 6：扫描优化项目**
```bash
mo optimize --dry-run
```

**步骤 7：展示优化方案并询问确认**
```
📋 系统优化计划：

- 重建系统数据库和缓存
- 重置网络服务
- 刷新 Finder 和 Dock
- 清理诊断和崩溃日志
- 重建 Launch Services 和 Spotlight 索引

是否执行优化？
- 确认优化
- 取消
```

**步骤 8：用户同意后执行**
```bash
mo optimize
```

**步骤 9：再次检查状态**
```bash
mo status
```

## 工作流 3：卸载应用

**触发条件**：用户需要完全卸载某个应用

### 完整执行步骤

**步骤 1：确认应用存在**
```bash
ls /Applications | grep -i "应用名"
ls ~/Applications | grep -i "应用名"
```

**步骤 2：扫描卸载内容**
```bash
mo uninstall --dry-run
```

**步骤 3：展示卸载方案**
```
📋 卸载方案：[应用名称]

将删除：
- 应用程序本体：/Applications/[App].app
- 缓存文件：~/Library/Caches/[bundle-id]
- 偏好设置：~/Library/Preferences/[bundle-id]
- Application Support：~/Library/Application Support/[App]
- Logs：~/Library/Logs/[App]
- LaunchAgents（如有）

预计释放空间：XX.X GB

是否执行卸载？
- 确认卸载
- 取消
- 查看完整文件列表
```

**步骤 4：用户确认后执行**
```bash
mo uninstall
```
或交互式选择：
- 使用方向键导航
- 空格选择
- 回车确认

**步骤 5：报告结果**
```
✅ 卸载完成！

已删除应用：[应用名称]
清理关联文件：XX 个
释放空间：XX.X GB
```

## 工作流 4：清理开发项目

**触发条件**：用户需要清理 node_modules、target 等构建产物

### 完整执行步骤

**步骤 1：配置扫描路径**（首次使用）
```bash
mo purge --paths
```
或直接编辑 `~/.config/mole/purge_paths`

**步骤 2：扫描构建产物**
```bash
mo purge --dry-run
```

**步骤 3：展示扫描结果**
```
📊 项目构建产物扫描结果：

┌─────────────────────┬───────────┬─────────────┐
│ 项目名称             │ 大小       │ 类型         │
├─────────────────────┼───────────┼─────────────┤
│ my-react-app        │ 3.2 GB    │ node_modules│
│ rust-backend        │ 2.8 GB    │ target      │
│ python-api          │ 1.5 GB    │ venv        │
│ next-blog           │ 1.9 GB    │ node_modules│
├─────────────────────┼───────────┼─────────────┤
│ 总计                 │ 9.4 GB    │             │
└─────────────────────┴───────────┴─────────────┘

⚠️ 注意：7天内的项目标记为 [Recent]

是否清理以上构建产物？
- 确认清理
- 取消
- 仅清理部分项目
```

**步骤 4：用户确认后执行**
```bash
mo purge
```

**步骤 5：报告结果**
```
✅ 清理完成！

已清理项目：X 个
释放空间：XX.X GB
```

## 工作流 6：综合维护

**触发条件**：用户要求系统维护、全面清理

### 完整执行步骤

**步骤 1：系统状态检查**
```bash
mo status
mo --version
df -h /
```

**步骤 2：依次扫描各类项目**

2.1 扫描系统缓存
```bash
mo clean --dry-run
```
→ 展示结果 → 询问确认 → 执行

2.2 扫描安装包
```bash
mo installer --dry-run
```
→ 展示结果 → 询问确认 → 执行

2.3 扫描构建产物
```bash
mo purge --dry-run
```
→ 展示结果 → 询问确认 → 执行

2.4 扫描优化项目
```bash
mo optimize --dry-run
```
→ 展示结果 → 询问确认 → 执行

**步骤 3：生成维护报告**
```
📋 系统维护报告

┌─────────────────────┬───────────┐
│ 操作                 │ 释放空间   │
├─────────────────────┼───────────┤
```

## 工作流 7：检查大文件

**触发条件**：用户需要找出占用空间的大文件

**注意**：`mo analyze` 是只读操作，安全，但仍需用户确认删除。

### 完整执行步骤

**步骤 1：启动磁盘分析**
```bash
mo analyze
```

**步骤 2：告知用户交互方式**
```
已启动磁盘分析工具。

操作说明：
- ↑↓ 或 j/k：导航
- L：查看大文件列表
- O：在 Finder 中打开
- ⌫ + Enter：删除到废纸篓（需确认）
- Q：退出

文件将移到废纸篓，可通过 Finder 恢复。
```

## 工作流 8：安装/更新 Mole

**触发条件**：Mole 未安装或需要更新

### 安装步骤
```bash
# Homebrew 安装
brew install tw93/tap/mole

# 验证安装
mo --version
```

### 更新步骤
```bash
# 更新到最新稳定版
mo update

# 强制重装
mo update --force

# 安装开发版
mo update --nightly
```

## ✅ 每次操作前的检查清单

在执行任何破坏性操作前，**必须**完成：

- [ ] 运行 `--dry-run` 扫描
- [ ] 以清晰格式向用户展示扫描结果
- [ ] 说明将要删除的内容和大小
- [ ] 使用 AskUserQuestion 询问用户确认
- [ ] **收到用户明确同意**后才执行
- [ ] 执行后报告结果

## ❌ 绝对禁止

- 未经扫描直接执行 `mo clean`
- 未经用户确认执行任何删除操作
- 假设用户同意而跳过确认步骤
- 在用户不知情的情况下清理文件
- 忽略用户白名单配置
- 在 Time Machine 备份期间强制清理
