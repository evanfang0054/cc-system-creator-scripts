#### 请求信息

**方法：** `[GET/POST/PUT/DELETE]`

**端点：** `/path/to/endpoint`

**认证：** 是/否

#### 请求参数

##### Query 参数

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| param1 | string | 是 | - | [参数说明] |
| param2 | number | 否 | 10 | [参数说明] |

##### Body 参数（如适用）

\```json
{
  "field1": "value1",
  "field2": "value2"
}
\```

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| field1 | string | 是 | [字段说明] |
| field2 | number | 否 | [字段说明] |

#### 请求示例

\```http
[METHOD] /path/to/endpoint?param1=value1
Host: api.example.com
Authorization: Bearer [your-token]
Content-Type: application/json

{
  "field1": "value1",
  "field2": "value2"
}
\```

#### 响应格式

##### 成功响应

**状态码：** `200 OK`

\```json
{
  "code": 200,
  "message": "success",
  "data": {
    "result": "data"
  }
}
\```

| 字段名 | 类型 | 说明 |
|--------|------|------|
| code | number | 响应状态码 |
| message | string | 响应消息 |
| data | object | 响应数据 |

##### 错误响应

**状态码：** `400 Bad Request`

\```json
{
  "code": 400,
  "message": "error message",
  "errors": [
    {
      "field": "param1",
      "message": "error detail"
    }
  ]
}
\```

#### 使用示例

\```javascript
// JavaScript 示例
const response = await fetch('/path/to/endpoint?param1=value1', {
  method: '[METHOD]',
  headers: {
    'Authorization': 'Bearer [your-token]',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    field1: 'value1',
    field2: 'value2'
  })
});

const data = await response.json();
console.log(data);
\```

## 使用指南

### 选择合适的模板

1. **通用技术文档**：适用于大多数技术文档
2. **API 文档**：专门用于 API 接口文档
3. **用户手册**：面向最终用户的操作指南

### 自定义模板

根据具体需求，可以：

- 调整章节顺序
- 添加或删除章节
- 修改内容结构
- 更新格式样式

### 模板最佳实践

1. **保持一致性**：在整个文档中使用统一的格式和风格
2. **明确受众**：根据目标读者调整内容深度和表达方式
3. **提供示例**：包含实用的代码示例和使用场景
4. **定期更新**：保持文档与产品功能同步更新
5. **收集反馈**：根据用户反馈持续改进文档

## 格式规范

### 标题层级

- 一级标题（#）：文档标题
- 二级标题（##）：主要章节
- 三级标题（###）：子章节
- 四级标题（####）：细节说明

### 代码块

- 使用三个反引号包裹代码
- 指定编程语言
- 添加必要的注释

### 表格

- 使用标准 Markdown 表格格式
- 包含表头
- 对齐方式一致

### 强调

- **加粗**：重要内容
- *斜体*：术语或强调
- `代码`：技术术语或代码

### 列表

- 无序列表：使用 `-`
- 有序列表：使用数字
- 任务列表：使用 `- [ ]`

### 引用

- 使用 `>` 引用重要信息
- 可以添加表情符号增强视觉效果
  - 💡 提示
  - ⚠️ 警告
  - ❗ 重要
  - ✅ 成功
  - ❌ 错误
