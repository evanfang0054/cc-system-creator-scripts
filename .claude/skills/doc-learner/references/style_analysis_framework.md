# 文档风格分析框架

## 概述

本文档提供了一套系统化的文档风格分析框架，用于识别、理解和模仿各类文档的写作风格，确保生成内容的一致性和专业性。

## 风格维度分析模型

### 1. 语言维度

#### 1.1 词汇选择

**词汇复杂度：**
- **简单词汇：** 使用日常、普及的词汇
  - 示例："使用"、"帮助"、"简单"
- **专业术语：** 使用领域特定术语
  - 示例："实例化"、"封装"、"异步"
- **学术词汇：** 使用正式、学术性词汇
  - 示例："基于"、"通过"、"实现"

**词汇密度：**
- **低密度：** 使用简单的单词表达
  - "帮助用户完成任务"
- **高密度：** 使用更多的名词化表达
  - "协助用户实现任务目标"

**术语使用模式：**
- 术语频率（专业术语出现频率）
- 术语解释习惯（首次出现是否解释）
- 缩写使用（是否使用缩写及缩写规则）

#### 1.2 句式结构

**句子长度：**
- **短句：** 10 字以下
  - "点击按钮。"
- **中句：** 10-30 字
  - "点击页面上的按钮，开始下载过程。"
- **长句：** 30 字以上
  - "点击页面右上角的下载按钮，系统将自动开始下载安装程序到您的本地设备。"

**句式多样性：**
- 简单句："系统支持这个功能。"
- 并列句："系统支持这个功能，并提供配置选项。"
- 复合句："当系统检测到新数据时，会自动触发同步流程，并更新本地缓存。"

**句子结构：**
- 主动语态 vs 被动语态
- 陈述句 vs 祈使句 vs 疑问句
- 倒装句 vs 正常语序

#### 1.3 语气特征

**语气类型：**
- **客观中性：** 事实陈述，无情感色彩
  - "系统版本为 1.0.0"
- **专业权威：** 专业知识，可信度高
  - "根据最佳实践，建议..."
- **友好亲和：** 接近口语，易于理解
  - "让我们来学习如何..."
- **简洁直接：** 简练表达，直击要点
  - "点击确定。"

**语气标记：**
- 使用的副词和形容词
- 情感词汇的使用
- 修饰语的使用频率

### 2. 结构维度

#### 2.1 组织模式

**层次结构：**
- **扁平结构：** 少量层级，信息平铺
  - 标题 → 内容
- **层次结构：** 多级标题，详细分类
  - 标题 → 章节 → 小节 → 要点

**信息流：**
- **线性：** 按时间或步骤顺序
- **层级：** 从总体到细节
- **网状：** 多维度关联

**段落组织：**
- 段落长度（字数）
- 段落功能（引入、展开、总结）
- 段落过渡方式

#### 2.2 章节划分

**标题风格：**
- 名词性标题："系统架构"
- 动词性标题："配置环境"
- 问题性标题："如何部署？"
- 陈述性标题："系统支持的功能"

**章节长度：**
- 短章节（< 200 字）
- 中等章节（200-500 字）
- 长章节（> 500 字）

#### 2.3 逻辑关系

**连接词使用：**
- 顺承："首先"、"其次"、"然后"
- 转折："但是"、"然而"、"不过"
- 因果："因此"、"所以"、"由于"
- 递进："此外"、"而且"、"更进一步"

**论证方式：**
- 归纳：从例子到结论
- 演绎：从原理到应用
- 类比：用比喻说明
- 对比：通过比较突出特点

### 3. 格式维度

#### 3.1 标点使用

**标点偏好：**
- 逗号使用频率
- 句号 vs 顿号
- 引号类型（单引号 vs 双引号）
- 破折号使用

**特殊符号：**
- 项目符号（•、-、*）
- 编号（1.、1)、（1））
- 箭头（→、➜）
- 其他符号（※、▶、▸）

#### 3.2 强调方式

**强调手段：**
- **加粗：** 重点内容
- *斜体：* 术语或注释
- `代码：` 技术术语
- > 引用：重要说明

**强调频率：**
- 高频：每段都有强调
- 中频：重要内容强调
- 低频：很少使用强调

#### 3.3 列表使用

**列表类型：**
- 无序列表
- 有序列表
- 任务列表
- 定义列表

**列表深度：**
- 单层列表
- 二层列表
- 多层嵌套

### 4. 内容维度

#### 4.1 详细程度

**信息密度：**
- **低密度：** 详细说明，多处举例
- **中密度：** 适度说明，关键处举例
- **高密度：** 精炼表达，少举例

**细节程度：**
- 包含实现细节
- 包含使用场景
- 包含边界情况
- 包含历史背景

#### 4.2 示例使用

**示例类型：**
- 代码示例
- 场景示例
- 数据示例
- 图表示例

**示例呈现：**
- 内联示例：嵌入文字中
- 独立示例：单独代码块
- 详细示例：带注释和说明

#### 4.3 参考引用

**引用方式：**
- 超链接
- 文档引用
- 书籍引用
- 规范引用

**引用频率：**
- 频繁引用
- 偶尔引用
- 很少引用

## 风格特征识别方法

### 步骤 1：整体感知

快速浏览文档，形成初步印象：

**问题引导：**
- 这是什么类型的文档？
- 给谁看的？
- 什么场景下使用？
- 给人的整体感觉是什么？

**记录第一印象：**
- 正式/非正式
- 专业/通俗
- 简洁/详细
- 冷淡/友好

### 步骤 2：系统分析

按照维度模型系统分析：

**2.1 语言分析**
- 提取关键词汇
- 统计句式类型
- 识别语气特征

**2.2 结构分析**
- 绘制文档结构图
- 分析章节组织
- 识别逻辑关系

**2.3 格式分析**
- 记录标点使用
- 统计强调方式
- 分析列表使用

**2.4 内容分析**
- 评估信息密度
- 统计示例数量
- 检查引用方式

### 步骤 3：模式提炼

总结风格模式：

**核心特征：**
- 最显著的风格特点
- 最常用的表达方式
- 最独特的格式习惯

**风格标签：**
- [ ] 正式/非正式
- [ ] 专业/通俗
- [ ] 简洁/详细
- [ ] 客观/主观
- [ ] 理论/实践
- [ ] 学术/实用

**风格示例：**
收集典型句子或段落作为风格参考

### 步骤 4：风格描述

用简洁的语言描述风格：

**描述框架：**
```
这是一篇[特征1]、[特征2]的[文档类型]。

语言方面：[描述语言特征]
结构方面：[描述结构特征]
格式方面：[描述格式特征]

整体风格：[总结整体印象]
适用场景：[适用场景和对象]
```

## 风格迁移技巧

### 1. 词汇迁移

**词汇替换：**
- 简单 → 专业："用" → "使用/调用/执行"
- 专业 → 简单："实例化" → "创建"
- 普通 → 学术："可以" → "支持/允许"

**术语统一：**
- 识别术语集
- 建立术语映射
- 批量替换

### 2. 句式迁移

**句式转换：**
- 短句 → 长句：添加修饰和细节
- 长句 → 短句：拆分简化
- 主动 → 被动：调整语态
- 被动 → 主动：调整语态

**句式模仿：**
- 分析目标句式结构
- 提取句式模板
- 应用新内容

### 3. 语气迁移

**语气调整：**
- 识别语气标记
- 替换语气词汇
- 调整表达方式

**示例：**
- 客观 → 友好：添加礼貌用语
- 正式 → 非正式：简化表达
- 冷淡 → 热情：添加积极词汇

### 4. 格式迁移

**格式统一：**
- 统一标题层级
- 统一标点符号
- 统一列表风格
- 统一强调方式

**结构对齐：**
- 分析目标结构
- 调整内容顺序
- 添加缺失部分
- 删除冗余内容

## 一致性保持策略

### 1. 建立风格指南

创建明确的风格规范：

**词汇规范：**
- 术语表
- 禁用词列表
- 推荐词列表

**句式规范：**
- 句长要求
- 句式偏好
- 语态选择

**格式规范：**
- 标题样式
- 标点使用
- 强调方式

### 2. 使用模板

创建可复用模板：

**结构模板：**
- 文档框架
- 章节模板
- 段落模板

**句式模板：**
- 常用句型
- 固定搭配
- 过渡表达

### 3. 检查清单

建立一致性检查清单：

**语言一致性：**
- [ ] 术语使用统一
- [ ] 语气保持一致
- [ ] 句式风格统一

**结构一致性：**
- [ ] 层级结构统一
- [ ] 章节长度相近
- [ ] 逻辑关系一致

**格式一致性：**
- [ ] 标题格式统一
- [ ] 标点使用统一
- [ ] 列表风格统一

### 4. 迭代改进

持续优化一致性：

**收集反馈：**
- 读者反馈
- 同行评审
- 使用数据

**分析问题：**
- 识别不一致处
- 分析原因
- 制定改进方案

**持续优化：**
- 更新风格指南
- 完善模板
- 改进流程

## 风格对比分析

### 对比方法

**选择对比维度：**
- 关键维度对比
- 整体风格对比
- 局部特征对比

**制作对比表：**

| 维度 | 文档 A | 文档 B | 差异 |
|------|--------|--------|------|
| 词汇 | 简单 | 专业 | 中等 |
| 句式 | 短句 | 长句 | 较大 |
| 语气 | 友好 | 客观 | 较大 |
| 结构 | 扁平 | 层次 | 中等 |

**差异总结：**
- 主要差异点
- 相似之处
- 适配建议

### 风格融合

**融合策略：**
- 保留核心特征
- 借鉴优点
- 形成新风格

**融合方法：**
1. 分析两种风格
2. 识别兼容点
3. 设计融合方案
4. 验证效果

## 常见问题

### Q1: 如何处理风格冲突？

**A:** 优先考虑文档类型和受众，选择最匹配的风格。必要时可以融合多种风格的优点。

### Q2: 风格分析需要多深入？

**A:** 根据需求决定。快速分析关注主要特征，深度分析关注细节和模式。

### Q3: 如何确保风格一致性？

**A:** 建立风格指南，使用模板，定期检查，收集反馈持续改进。

### Q4: 可以创新风格吗？

**A:** 可以。在理解现有风格的基础上，根据具体需求和场景创造性地应用和调整。
